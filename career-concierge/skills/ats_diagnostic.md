## When to Use
Use this skill when evaluating technical resumes for high-tier infrastructure, systems, or distributed applications engineering tracks where claims must be rigidly verifiable and quantitative.

## Operational Playbook
1. Enforce Quantifiable Impact: Every technical bullet point must possess a measurable outcome (e.g., % latency reduction, throughput scaling factors, memory footprint savings).
2. Action-Verb Priming: Eliminate passive phrases like "responsible for helping to develop". Replace with strong systems-engineering active verbs: "Engineered", "Optimized", "Architected", "Profiled", "Refactored".
3. Anti-Patterns to Reject: 
   - Vague adjectives ("fast backend applications").
   - Clustered keyword-stuffing blocks without situational context.
